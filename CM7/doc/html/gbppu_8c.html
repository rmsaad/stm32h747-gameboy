Detailed Description

: Gameboy PPU Functionality This file emulates all functionality of the Gameboy Pixel/Picture Processing Unit (PPU).

This includes drawing the Background, Window, and Object Layers to a frame buffer to be displayed 60 times a second.

◆ gbPPUStep()

void gbPPUStep ( )

Steps the PPU by tStates.

This function steps the PPU by the tStates variable if the screen enable (7th) bit of LCDC Register is high, the function sets the mode in the STAT Register using the setMode() function depending on how many tStates have elapsed. after a full line is has passed the LY register is updated vGBPPUDrawLine() is called to place data in the frame buffer.

Returns
Nothing
Note
The function and the Gameboy both mimic CRT Displays in that their are both a horizontal H-Blank after a Line is "drawn" and there is a vertical V-Blank after a full frame is drawn
Attention
The placement of the functions vCheckBackWinTileDataSel() and vCheckBackTileDisplaySel() are just educated guesses at the current moment, more research into when and how often these Registers are updated must be conducted.

uint16_t usGetBackTileDisplaySel ( )

Read LCDC Register to find appropriate Background Tile Display Select Address.

This function Reads from the LCDC Register and depending on the value of the 3rd bit, it will change the Background Tile Display Address to 0X9C00 on high or to 0x9800 on low.

Returns

◆ usGetBackWinTileDataSel()

uint16_t usGetBackWinTileDataSel ( )

Read LCDC Register to find appropriate Background & Tile Data Address.

This function Reads from the LCDC Register and depending on the value of the 4th bit, it will change the Background & Window Tile Data Select Address to 0x8000 on high or to 0x8800 on low.

Returns
Nothing
Note
Background & Window Tile Data Select should be 0x9000 on a low 4th bit in a real Gameboy, but it is much simpler to implement this way with emulation, the 0x400 offset is accounted for in getTileLineData()
